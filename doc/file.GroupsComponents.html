Exporting Groups and Components

About groups and components

Groups and components might seem a similar thing. However, they are very different and they are actually treated differently by Groundhog.

A Group is, well, a group of elements that are tied together. You can group, for example, a set of luminaires or furniture. Then, you can copy and paste the whole group, move it around, delete it, etc. Groups, from the Grounhog point of view, are meant to simplify the process of creating and handling models, however, they are treated just as any other surface or line. Accordingly, they will be exported with each layer just as any other geometry.

Components, on the other hand, can be thought as groups that are meant to be repeated a number of times and that will always be equal. For example, an office may have 15 chairs, and they will all be the same. In this case, it makes little sense to export the same chair 15 times. On the contrary, it is better to export it just once and then tell Radiance that we are including that chair in different locations and orientations within the Office. Accordingly, Components will be exported just once in a different file and then included into the main 'scene.rad' file by using the Radiance's '!xform' command.

Nested components (components that lay within another component) will be “!xformed” inside the definition of the parent component (i.e. within the .rad file located in the corresponding directory). First level components (components that are not inside any other component), on the other hand, will be “!xformed” in the scene file. Play a little with this.

Why treating them differently

On one hand, exporting Components once instead of a number of times saves space and helps keeping everything in order and under control. On the other hand, this is very useful for, for example, exporting Shading Devices. Although this is a very specific personal interest, designing a Shading Device within SketchUp as a Component allows exporting it in its own file, making it easier to analyze it with Radiance. For example, placing it in the right orientation and position (within its own references), allows directly running Radiance's 'genBSDF' program, that might be used for implementing 3 or 5 phase methods.

Notes

Components within components, and components within groups seem to work fine. Groups within components, though, seem to be working incorrectly. Please avoid that by exploding (right click -> explode) the groups that lay inside other groups or components… I have found this very tricky to program. If you find a bug, please let me know.
